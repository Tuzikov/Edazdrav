# Приложение подсчёта калорий по фото — бриф проекта

## Идея
Android-приложение (кроссплатформенное, React Native/Expo): пользователь
фотографирует блюдо → приложение распознаёт ингредиенты и порции →
считает калории, БЖУ и клетчатку.

## Аудитория/контекст разработчика
Без опыта в программировании, ведём проект через Claude Code —
пошагово, с реальным запуском и тестированием на Android через Expo Go.

## Стек
- **Frontend**: React Native + Expo
- **Backend**: лёгкий прокси-сервер (Node.js, Supabase Edge Functions или
  Firebase Cloud Functions) — прячет API-ключи, дергает Gemini и USDA
- **Хранение**: Supabase (Postgres + auth) — история приёмов пищи, цели пользователя
- **Vision/AI**: Gemini API (бесплатный тариф на старте)
- **База нутриентов**: USDA FoodData Central (бесплатно, ключ уже получен)

## Пайплайн распознавания (ключевая логика)

1. Фото отправляется в бэкенд
2. Бэкенд вызывает Gemini с промптом, который просит вернуть JSON вида:
   ```json
   [{"name_ru": "куриная грудка гриль", "name_search": "grilled chicken breast", "grams": 150}]
   ```
   Оба названия генерируются за один проход — это важно, не два отдельных запроса.
3. Для каждого ингредиента бэкенд ищет `name_search` в USDA FoodData Central
   (приоритет типам данных Foundation и SR Legacy)
4. Достаём нутриенты на 100г (Energy, Protein, Total lipid (fat),
   Carbohydrate by difference, Fiber total dietary), масштабируем на граммовку
5. Суммируем по всем ингредиентам блюда

### Ручное добавление ингредиента (юзер вводит текст на русском)
Отдельного `name_search` нет — нужен промежуточный шаг:
небольшой текстовый запрос к Gemini "переведи название продукта на
английский для базы данных питания США" → дальше та же логика поиска в USDA.

## Экраны и навигация

Нижние вкладки: **Сегодня** / **История** / **Профиль**, плюс отдельный
модальный flow добавления блюда (не вкладка).

### 1. Профиль/цели (заполняется при первом запуске)
Поля: калории/день, белки (г), жиры (г), углеводы (г), клетчатка (г).
Значения по умолчанию: 2000 ккал, Б100/Ж65/У225, клетчатка 28г — пользователь
может поправить под себя. Редактируется в любой момент из вкладки Профиль.

### 2. Сегодня (главный экран)
- Кольцо прогресса по калориям (факт/цель)
- 4 карточки: белки/жиры/углеводы/клетчатка (факт/цель)
- Список приёмов пищи за день (иконка, время, калории)
- Кнопка добавления блюда (камера)

### 3. Добавление блюда (flow)
Камера/галерея → экран "Анализ" (спиннер, идёт запрос к Gemini) →
экран "Проверь результат":
  - фото сверху
  - список распознанных ингредиентов, каждый с полем граммовки (editable)
  - кнопка "Добавить ингредиент" (ручной ввод на русском + перевод под капотом)
  - итоговая строка калорий/БЖУ/клетчатки, пересчитывается на лету при правке
  - кнопка "Сохранить приём пищи" → назад на "Сегодня"

### 4. История
MVP: простой список приёмов пищи по дням (лента). Календарь — возможное
улучшение на будущее, не блокирует MVP.
