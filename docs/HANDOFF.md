## Что это за проект
Android-приложение подсчёта калорий по фото (React Native / Expo).
Бриф — [PROJECT_SPEC.md](../PROJECT_SPEC.md). Название и иконка — «Едазавр»
(T-Rex из еды).

## Текущее состояние — MVP полностью рабочий
- Вкладки Сегодня / История / Профиль, flow добавления блюда
  (фото / галерея / вручную), экран деталей и редактирования уже
  сохранённых приёмов.
- Реальный бэкенд: Supabase Edge Functions (`recognize-meal`,
  `lookup-nutrients`, `lookup-ingredient`) — дёргают Gemini (распознавание
  фото + перевод названий) и USDA FoodData Central (нутриенты). Gemini —
  платный тир (был лимит free tier 20 запросов/день, пользователь включил
  биллинг).
- Калькулятор целей (формула Миффлина-Сан Жеора) в Профиле, плюс ручной
  ввод целей.
- История: факт/план по калориям и выбранному нутриенту в шапке каждого
  дня; счётчик «N дней в режиме 👍» (геймификация, считает подряд идущие
  календарные дни без превышения плана по калориям).
- Кольцо калорий на «Сегодня» краснеет при перерасходе.
- Android: edge-to-edge поддержан (`react-native-edge-to-edge` +
  `useSafeAreaInsets`), иконка приложения и adaptive icon — с отступами под
  безопасную зону маски, без обрезки.
- Последний собранный и подтверждённый пользователем на реальном устройстве
  APK — коммит `e18d5f2`. Ссылки на артефакты EAS Build не вечные — если
  ссылка из истории чата уже не работает, просто пересобрать:
  `npx eas-cli build -p android --profile preview --non-interactive`.

## На заметку (грабли, на которые уже наступали)
- USDA FoodData Central API периодически отдаёт 400 без видимой причины
  (не связано с частотой запросов) — в `supabase/functions/_shared/usda.ts`
  уже есть ретрай (до 4 попыток), но полностью исключить нельзя.
- Общий фолбэк по нутриентам (120 ккал / 6Б / 5Ж / 12У / 1.5 клетч. на
  100г, см. `GENERIC_FALLBACK` там же) срабатывает, если USDA вообще не
  находит совпадение — может давать неточные цифры для редких/специфичных
  продуктов.
- Промпт для Gemini явно разделяет "твёрдая еда" (можно добавлять
  raw/grilled/boiled) и "напитки/жидкости" (нельзя добавлять raw/cooked) —
  раньше "вода, raw" уводила поиск USDA в сторону не той еды.
- Фото в истории хранятся как временная cache-ссылка от `expo-image-picker`
  (не копируются в постоянное хранилище) — при очистке кэша Android
  картинка может перестать открываться, но данные о калориях не пострадают.
  Обсуждали с пользователем, договорились пока не чинить.
- Локальный `~/.npm` кэш на машине пользователя один раз был испорчен
  root-owned файлами — если `npm install` внезапно падает с `EACCES`:
  `sudo chown -R $(whoami):staff ~/.npm`.
- На macOS для растровых операций с иконками используется `sips` (встроен)
  и Pillow (`pip3 install Pillow`) — оба уже проверялись рабочими на этой
  машине.
